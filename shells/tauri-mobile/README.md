# Tauri Mobile Shell

iOS and Android distribution via Tauri 2. Shares the web shell source and state bridge override with the desktop shell; differs in capabilities, identifier, and the platform-specific `gen/` projects generated by Tauri's mobile init commands.

## Feature subset

Tools that declare `capabilities: ["filesystem", "ffmpeg", "wasm"]` are not available on mobile — the gallery filters them. Declarative tools (no sidecar requirements) work in full. Camera-enabled tools work via the WebView's `getUserMedia` API.

## What's different from the desktop shell

| Concern | Desktop | Mobile |
|---|---|---|
| Identifier | `com.suse.brand-tools` | `com.suse.brand-tools.mobile` |
| Shell plugin | ✅ (sidecars) | ❌ |
| Camera | ❌ | Via WebView getUserMedia |
| Dev port | 5173 | 5174 |
| Platform targets | macOS / Windows / Linux | iOS / Android |

## Structure

```
shells/tauri-mobile/
├── package.json
├── vite.config.js            # same override pattern as desktop
├── bridge-overrides/
│   └── state.js              # filesystem state (same API as desktop)
├── dist/                     # Vite output (gitignored)
├── gen/                      # generated by tauri android/ios init (gitignored)
└── src-tauri/
    ├── Cargo.toml
    ├── build.rs
    ├── tauri.conf.json
    ├── capabilities/
    │   └── default.json
    └── src/
        ├── main.rs
        └── lib.rs
```

## First-time setup

Platform init is a one-time step that generates the `gen/` directory:

```bash
cd shells/tauri-mobile
npm install

# Android
npm run tauri android init

# iOS (macOS only)
npm run tauri ios init
```

## Dev

```bash
# Android (emulator or connected device)
npm run dev:android

# iOS simulator (macOS only)
npm run dev:ios
```

## Build

```bash
npm run build:android   # outputs APK + AAB to gen/android/
npm run build:ios       # outputs .ipa to gen/apple/
```

See `docs/build-guide.md` for full prerequisite setup (Android Studio, Xcode, signing).
