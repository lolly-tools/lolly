# brands/lolly-start — the blank starter brand

The brand pack you get when you don't have one: a near-empty catalog (a single
neutral starter-tokens asset — no fonts, no other assets) merged with the
community tools from `community/`. It is
parent-owned (not a submodule) so anyone who clones the public monorepo — with
no access to the private `brands/suse` pack — lands on a working profile:
`scripts/use-profile.ts --auto` falls back to it automatically.

Activate explicitly with:

```bash
npm run profile:start     # tools/ = community only, catalog → here
```

`catalog/tools/index.json` is generated by `npm run build:catalog` (run while
this profile is active) and committed like any other build artifact of the
parent repo.

## Neutral starter tokens, and hydrating a real brand

The web shell's token bridge doesn't hardcode SUSE's asset id — it discovers
the first catalog asset with `type: "tokens"` generically, whichever profile
is active. Here that's `catalog/assets/lolly/tokens/brand.json`: a small,
hand-editable DTCG document of neutral placeholder colours (blacks/whites/
grays plus a generic chart-accent spectrum), so the colour picker has real
swatches to resolve instead of silently falling back to hardcoded ones.

To replace it with an actual brand, run:

```bash
npm run ingest:brand -- <tokens.json | token-sets-dir | file.penpot> --name <brand> --register
```

`scripts/ingest-brand.ts` accepts a monolithic Tokens-Studio/DTCG JSON file, a
Penpot multi-file token-set export (a directory), or a raw `.penpot` project
file, and hydrates a new brand pack from it — replacing this starter tokens
file wholesale.
